iptables 命令解释:

iptables [AI链名] [-io 网络接口] [-p 协议] [-s 来源IP/网域] [-d 目标IP/网域] -j[ACCEPT DROP REJECT LOG]

选项与参数:  
    -AI链名 针对某链进行规则的 添加 或 累加
        -A ：新增加一条规则，该规则增加在原本规则的最后面
        -I ：插入一条规则，如果没有指定此规则的顺序，默认是插入变成第一条规则。
    -io网络接口:设定封包进出的接口
        -i: 封包所进入的某网络接口 eth0,lo 等,配合IUPUT链
        -o: 封包所传出的某网络接口, 配合OUTPUT链

    -p协议: tcp udp icmp ,all

    -s来源IP/网域     
        IP  ：192.168.0.100
        网域：192.168.0.0/24, 192.168.0.0/255.255.255.0 均可。
        若规范为"不允许"时，则加上 ! 即可，例如：
        -s ! 192.168.100.0/24 表示不许 192.168.100.0/24 之封包来源；

    -d目标IP/网域 同 -s

    -j 动作(Action)
        接受(ACCEPT)、丢弃(DROP)、拒绝(REJECT)及记录(LOG)


针对端口(port)的设定:
    针对端口的防火墙设定,需要确定 **-p** 参数,也就是需要确定使用的什么协议
    iptables -A  INPUT -i eth0 -p tcp --dport 8800 -j REJECT    
    参数: 
        --sport: 来源端口,端口号可以是连续的:1024:65535
        --dport: 目标端口.可以连续写法同上
    **如果使用 --sport/--dport 参数时,必须在前面加 -p 协议 参数 否则 报错:unkown option**    
    TCP中特殊标识的处理:SYN(主动联机)
    iptables -A INPUT -i eth0 -p tcp --sprot 1:1023 --dport 1:1023 --syn -j DROP


针对ICMP(Internet Control Message Protocol 网络消息控制协议)封包的规则
    iptables -A INPUT [-p icmp] [icmp--type 类型] -j ACCEPT
    --icmp-type: ICMP的封包类型
    类别代号    名称/含义
    0            Echo Reply (代表一个回应信息)
    3            Destination Unreachable (表示目的地不可到达)
    4            Source Quench (当 router 的负载过高时，此类代码可以用来让发送端停止发送消息)
    5            Redirect (重定向路由)
    8            Echo Request (请求回应信息)
    11            Time Exceeded for a Datagram (当封包在路由传送中超时，通知来源该封包已经被忽略)
    12            Parameter Problem on a Datagram (当一个ICMP封包重复之前的错误时，会回复来源主机参数错误的信息)
    13            Timestamp Request (要求对方发送时间信息，用以计算路由时差，以满足同步性协定的需求)
    14            Timestamp Reply (回应 Timestamp Request 用)
    15            Information Request (在 RARP 协定应用之前，此信息是用來在开机时取得网路信息)
    16            Information Reply (用以回应 Infromation Request 信息)
    17            Address Mask Request (用来查询子路由 mask 设定信息)
    18            Address Mask Reply (回应子路由 mask 查询信息)

设定防火墙的步骤:

1.规则归零：清除所有已经存在的规则 (iptables -F...)
        选项与参数：
            -F ：清除所有的已订定的规则；
            -X ：杀掉所有使用者 "自定义" 的 chain (应该说的是 tables ）啰；
            -Z ：将所有的 chain 的计数与流量统计都归零
2.预设政策：除了 INPUT 这个自定义链设为 DROP 外，其他为预设 ACCEPT；
        选项与参数：iptables [-t nat] -P [INPUT,OUTPUT,FORWARD] [ACCEPT,DROP]
            -P ：定义政策( Policy )。
                ACCEPT ：该封包可接受
                DROP   ：该封包直接丢弃，不会让 client 端知道为何被丢弃。
3.信任本机：由于 lo 对本机来说是相当重要的，因此 lo 必须设定为信任装置；
4.回应封包：让本机主动向外要求而响应的封包可以进入本机 (ESTABLISHED,RELATED)
5.信任用户：这是非必要的，如果你想要让区网的来源可用你的主机资源时

脚本:(xx.sh)

#!/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin; export PATH

# 1. 清除规则
    iptables -F
    iptables -X
    iptables -Z
# 2. 设定政策
    iptables -P   INPUT DROP
    iptables -P  OUTPUT ACCEPT
    iptables -P FORWARD ACCEPT
# 3~5. 制订各项规则
    iptables -A INPUT -i lo -j ACCEPT
    iptables -A INPUT -i eth0 -m state --state RELATED,ESTABLISHED -j ACCEPT
    #iptables -A INPUT -i eth0 -s 192.168.1.0/24 -j ACCEPT
# 6. 写入防火墙规则配置文件
    /etc/init.d/iptables save
